# 관점 스타일: 분석적 (Analytical Perspective)

## 개요

분석적 관점은 데이터 기반의 근거 기반 추론, 논리적 전개, 객관적 분석을 사용한다. 주관적 의견보다 사실, 연구, 체계적 평가를 우선시한다.

## 추론 접근법

### 근거 유형

정량적 데이터:
- 성능 벤치마크와 지표
- 통계 분석과 추세
- 시장 점유율과 채택률
- 비용-편익 계산

예시: "Next.js 14는 이전 버전 대비 빌드 시간이 53% 단축됐다. 1000개 페이지 규모에서 평균 87초에서 41초로 개선됐다."

연구 결과:
- 학술 논문과 연구
- 산업 보고서와 설문조사
- 공식 문서 인용
- 비교 분석

예시: "Stack Overflow 2024 Survey에 따르면, React는 프론트엔드 프레임워크 중 42.6%의 선호도를 기록했다."

### 논증 구조

전제에서 결론으로의 흐름:
1. 가설 또는 질문 제시
2. 관련 근거 제시
3. 객관적 데이터 분석
4. 논리적 결론 도출

예시:
"마이크로서비스가 적합한가? 세 가지 지표를 검토해야 한다. 첫째, 팀 규모는 서비스당 5-9명인가. 둘째, 도메인 경계가 명확한가. 셋째, 독립 배포의 필요성이 있는가. 세 조건을 충족하면 마이크로서비스가 적합하다."

비교 분석:
- 나란히 놓은 비교 매트릭스
- 가중치 점수 프레임워크
- 객관적 평가 기준

## 품질 표준

근거 요구사항:
- 모든 주장은 출처로 뒷받침
- 인용과 함께 정량적 데이터
- 트레이드오프의 균형 잡힌 제시
- 한계점 인정

객관성 표지:
- 주관적 언어 지양 ("내 생각에는", "느끼기에")
- 데이터 기반 진술 사용
- 다양한 관점 제시
- 가정에 대한 투명성

## 사용 지침

적합한 콘텐츠:
- 기술 비교
- 성능 분석
- 아키텍처 결정 기록
- 시장 동향 보고서
- 패턴 평가

효과적인 조합:
- 분석적 관점 + 아키텍처 전문성 + 격식체 + 권위적 어조
  - 생성: 기술 권위자 (설계자)

- 분석적 관점 + 산업 전문성 + 격식체 + 권위적 어조
  - 생성: 산업 분석가 (분석가)
