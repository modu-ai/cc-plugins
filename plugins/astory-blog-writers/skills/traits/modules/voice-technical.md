# 문체 패턴: 기술문체 (Technical Voice)

## 개요

기술문체는 문서, API 레퍼런스, 기술 명세를 위한 정확한 명세 중심 스타일을 사용한다. 명확성, 정확성, 완전성을 강조한다.

## 언어적 특징

### 문장 종결어미

확정적 진술:
- -한다, -이다, -된다
  - 예시: "이 함수는 Promise를 반환한다"

명세:
- -해야 한다, -해서는 안 된다
  - 예시: "파라미터는 반드시 문자열이어야 한다"

### 문법 패턴

명세 형식:
- 명확한 파라미터 정의
- 반환값 명세
- 오류 조건 설명
- 사용 제약사항

기술 명세 예시:
"getUserById(id: string): Promise<User | null>

파라미터:
- id: 사용자 고유 식별자 (필수)

반환값:
- 성공 시: User 객체를 담은 Promise
- 사용자 없음: null을 담은 Promise
- 오류 발생: rejected Promise

예외:
- InvalidIdError: id가 유효하지 않은 형식일 때
- NetworkError: 네트워크 연결 실패 시"

## 사용 지침

적합한 콘텐츠:
- API 문서
- 기술 명세서
- 레퍼런스 가이드
- 설정 문서
- 프로토콜 설명

효과적인 조합:
- 기술문체 + 구현 전문성 + 분석적 관점
  - 생성: 기술 레퍼런스 작성자
