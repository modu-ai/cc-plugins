# 문체 패턴: 격식체 (Formal Voice)

## 개요

격식체는 한국어 선언문 형식의 격식 스타일(한다체)을 사용하며, 권위적이고 분석적인 어조와 복잡한 문장 구조가 특징이다. 이는 한국어 학술 및 기술 문서의 전통적인 문체다.

## 언어적 특징

### 문장 종결어미

평서문:
- 현재 시제: -다, -한다
  - 예시: "시스템은 이벤트를 처리한다"

- 과거 시제: -했다, -였다
  - 예시: "분석 결과는 예상과 일치했다"

- 미래/당위: -할 것이다, -해야 한다
  - 예시: "개발자는 이 패턴을 고려해야 한다"

의문문:
- -하는가, -인가
  - 예시: "이 접근법이 적절한가"

청유문:
- -하자, -해보자
  - 예시: "다음 예제를 살펴보자"

### 문법 패턴

복문 구성:
- 종속절을 광범위하게 사용
- 접속사를 통해 여러 아이디어를 연결
- 내포절을 활용한 계층적 정보 구성

복문 예시:
"마이크로서비스 아키텍처는 각 서비스가 독립적으로 배포되고 확장될 수 있다는 점에서 모놀리식 구조와 근본적으로 다르며, 이는 개발 팀의 자율성을 높이는 동시에 시스템 복잡도를 증가시킨다."

복합적 사고를 위한 접속사:
- 그러나 (역접)
- 반면에 (대조)
- 따라서 (인과)
- 이에 따라 (귀결)
- 뿐만 아니라 (첨가)
- 특히 (강조)
- 근본적으로 (본질)

### 단락 구조

도입부 패턴:
- 주요 개념을 명시하는 주제문으로 시작
- 확정적 언어로 권위 확립
- 예시: "이벤트 소싱은 시스템의 모든 상태 변화를 순차적 이벤트로 기록하는 아키텍처 패턴이다."

전개부 패턴:
- 각 문장이 이전 문장을 기반으로 발전
- 전환어를 사용하여 논리적 흐름 표현
- 3인칭으로 분석적 거리 유지

결론부 패턴:
- 주요 논점 종합
- 실행 가능한 결론 제시
- 예시: "결론적으로, 패턴의 적용 범위와 한계를 명확히 이해하고 적용해야 한다."

## 어휘 선택

격식 기술 용어:
- 정확한 기술 용어 사용
- 최초 사용 시 한글 + 영문 병기
- 예시: "일관성(consistency)"

전환 표현:
- 일상적 표현 대신 학술적 접속어 사용
- 예시:
  - "이를 위해" (목적)
  - "그 결과" (결과)
  - "즉" (환언)
  - "한편" (병렬)

비격식 표현 지양:
- -요 같은 대화체 어미 사용 금지
- 구어체 축약 표현 사용 금지
- "그냥" 같은 비격식 표현 사용 금지

## 어조와 문체 수준

권위 확립:
- 유보 없이 확정적으로 사실 기술
- 일관되게 평서법 사용
- 정확한 전문 용어로 전문성 표현

객관성 유지:
- 전체적으로 3인칭 시점 유지
- 인칭 대명사(나, 저, 당신) 사용 지양
- 의견이 아닌 보편적 관찰로 구성

분석적 거리:
- 독자와 전문적 거리 유지
- 정보를 확립된 지식으로 제시
- 적절한 경우 피동태 사용

## 사용 지침

적합한 콘텐츠:
- 기술 심층 분석 및 아키텍처 가이드
- 분석적 글 및 패턴 설명
- 학술적 기술 문서
- 권위 있는 입장문
- 객관적 기준에 따른 비교 분석

부적합한 콘텐츠:
- 초보자 대상 가벼운 튜토리얼
- 개인 경험 이야기
- 대화체 블로그 글
- 단계별 가이드(매우 격식적인 맥락 제외)

효과적인 조합:
- 격식체 + 아키텍처 전문성 + 분석적 관점 + 권위적 어조
  - 생성: 기술 권위자 페르소나 (설계자)

- 격식체 + 산업 전문성 + 분석적 관점 + 권위적 어조
  - 생성: 산업 분석가 페르소나 (분석가)

- 격식체 + 교육 전문성 + 분석적 관점 + 양육적 어조
  - 생성: 격식적 접근의 체계적 교육자

## 예시 패턴

도입부 훅 (격식체):
"분산 시스템에서 데이터 일관성은 CAP 정리의 제약 속에서 해결해야 할 핵심 과제다."

설명 패턴:
"이 접근의 핵심 장점은 세 가지다. 첫째, 완벽한 감사 추적이다. 둘째, 시계열 분석이다. 셋째, 시스템 회복이다."

결론 패턴:
"따라서 이 패턴은 서비스 특성과 도메인 복잡도를 면밀히 검토한 후 적용해야 한다."

## 흔한 실수

격식 수준 혼용:
- 잘못: "이 패턴은 좋아요" (격식과 비격식 혼용)
- 올바름: "이 패턴은 우수하다" (일관된 격식체)

유보적 표현:
- 잘못: "아마도 효과적일 것 같다" (격식체에 부적합)
- 올바름: "효과적이다" (확정적)

비격식 표현:
- 잘못: "그냥 이렇게 하면 된다" (너무 비격식)
- 올바름: "다음과 같이 구현한다" (격식체)

1인칭/2인칭 사용:
- 잘못: "우리는 이것을 본다" (1인칭 복수)
- 올바름: "이것은 명확하다" (3인칭 객관)
