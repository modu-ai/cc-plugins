# 크루 협업 역할 (Crew Collaboration Roles)

## 빠른 참조

### 목적

크루 역할 스킬은 크루 모드를 사용할 때 포괄적이고 고품질 콘텐츠 생성을 위한 다중 페르소나 협업을 조율하는 7개의 전문 조율 역할을 정의한다.

### 7개의 크루 역할

크리에이티브 디렉터 (Creative Director):
- 전체 콘텐츠 비전과 서사 아크 감독
- 구조와 접근법에 대한 최종 결정
- 다중 목소리 기여에도 일관된 스토리 보장

리서치 애널리스트 (Research Analyst):
- 기술적 정확성과 소스 신뢰성 검증
- 주장에 대한 데이터와 증거 제공
- 모든 페르소나 기여 사실 확인

리드 라이터 (Lead Writer):
- 섹션에 페르소나 할당 조율
- 여러 목소리에 걸친 스타일 일관성 유지
- 페르소나 전환 간 전환 처리

테크니컬 리뷰어 (Technical Reviewer):
- 코드 예시와 기술 세부사항 검증
- 구현 가이던스의 정확성 보장
- 아키텍처 권고 검증

리더 애드보케이트 (Reader Advocate):
- 대상 청중 관점 대변
- 혼란스러운 섹션이나 지식 격차 지적
- 청중 수준에 맞는 접근성 보장

데블스 애드보케이트 (Devil's Advocate):
- 가정과 기존 사고에 도전
- 잠재적 반론 식별
- 더 깊은 정당화를 강제하여 콘텐츠 강화

에디터 인 치프 (Editor-in-Chief):
- 최종 품질 검토 수행
- 글쓰기 표준 준수 강화
- 발행 결정 수행

### 크루 역할 사용 시점

크루 역할 사용 시점:
- 콘텐츠가 여러 기술 도메인에 걸쳐 있을 때
- 품질이 여러 관점에서의 교차 검증을 요구할 때
- 단일 포괄적 문서에 대한 팀 협업 시
- 엄격한 검토가 필요한 높은 위험 콘텐츠

크루 역할 건너뛰기 시점:
- 단순한 솔로 모드 콘텐츠로 충분할 때
- 시간 제약이 조율 오버헤드를 허용하지 않을 때
- 주제가 여러 전문가 관점에서 이익을 얻지 않을 때

## 구현 가이드

### 크리에이티브 디렉터 역할

크리에이티브 디렉터는 모든 페르소나 기여가 통합된 콘텐츠 비전에 기여하도록 전략적 감독을 제공한다.

책임:

비전 설정:
- 콘텐츠 목표와 대상 결과 정의
- 서사 구조와 흐름 확립
- 적절한 톤과 스타일 결정
- 섹션별 페르소나 할당 승인

품질 보증:
- 완료된 섹션의 비전 정렬 검토
- 콘텐츠 커버리지 격차 식별
- 섹션이 목표를 놓쳤을 때 수정 요청
- 부분들에서 일관된 전체가 나오도록 보장

결정 권한:
- 구조적 변경에 대한 최종 결정권
- 페르소나 할당 승인
- 역할 간 충돌 해결
- 발행 준비 상태 결정

일반적 할당:
- 종종 애널리스트 또는 아키텍트 페르소나가 담당
- 큰 그림 사고와 분석 기술 필요
- 의사결정을 위한 권위적 톤에서 이점

### 리서치 애널리스트 역할

리서치 애널리스트는 모든 사실적 주장이 정확하고, 출처가 있으며, 신뢰할 수 있도록 보장한다.

책임:

소스 검증:
- 인용된 모든 소스의 정확성과 최신성 검증
- URL이 유효하고 관련 있는 콘텐츠를 반환하는지 확인
- 권위 있는 소스에서 통계와 벤치마크 검증
- 증거가 필요한 지원되지 않는 주장 지적

기술 사실 확인:
- 공식 문서와 대조하여 기술 세부사항 확인
- 버전 번호와 릴리스 날짜 검증
- API 시그니처와 코드 구문 검증
- 논쟁적 주장에 대해 여러 소스 교차 참조

인용 관리:
- 전체적으로 적절한 인용 형식 보장
- 참조 번호가 인용과 일치하는지 확인
- 완전한 참조 섹션 편집
- 모든 자료의 소스 메타데이터 추적

일반적 할당:
- 애널리스트 페르소나가 자연스럽게 적합 (데이터 기반, 분석적)
- 리뷰어 페르소나도 적합 (균형 잡힌 평가)
- 세부사항에 대한 체계적인 주의 필요

### 리드 라이터 역할

리드 라이터는 스타일적 일관성과 부드러운 전환을 보장하며 다중 페르소나 협업을 조율한다.

책임:

섹션 할당:
- 콘텐츠 섹션을 적절한 페르소나에 매핑
- 콘텐츠 필요에 전문 영역 매칭
- 페르소나 간 작업량 균형
- 명확한 경계가 있는 할당 명세 생성

전환 관리:
- 섹션 간 명시적 전환 표지 설계
- 목소리를 연결하는 브리징 문장 작성
- 목소리 변화에도 서사 흐름 유지
- 전환에 걸쳐 독자 이해 보장

스타일 일관성:
- 모든 페르소나에 걸쳐 글쓰기 표준 강화
- 일관된 용어 사용 보장
- 적절한 격식 수준 유지
- 인용 스타일 균일성 검증

일반적 할당:
- 멘토 페르소나가 적합 (교육적 조율)
- 개발자 페르소나도 적합 (실용적 조직)
- 모든 페르소나 목소리에 대한 강한 이해 필요

### 테크니컬 리뷰어 역할

테크니컬 리뷰어는 정확성과 완전성을 위해 모든 기술 콘텐츠를 검증한다.

책임:

코드 검토:
- 모든 코드 예시가 올바르게 실행되는지 확인
- 대상 언어/프레임워크 버전의 구문 검사
- 예시가 모범 사례를 따르는지 확인
- 코드가 주장된 동작을 보여주는지 테스트

기술적 정확성:
- 아키텍처 패턴이 올바르게 설명되었는지 검증
- 시스템 설계 원칙의 정확성 확인
- 프레임워크 사용이 문서와 일치하는지 확인
- 구성 예시가 작동하는지 검증

구현 가이던스:
- 단계별 지침이 완전한지 확인
- 명령어와 설정 절차 검증
- 튜토리얼이 예상 결과로 이어지는지 확인
- 누락된 종속성이나 선수 지식 지적

일반적 할당:
- 개발자 페르소나가 이상적 (실습 검증)
- 아키텍트 페르소나도 적합 (설계 검증)
- 리뷰어 페르소나도 적합 (체계적 평가)

### 리더 애드보케이트 역할

리더 애드보케이트는 대상 청중을 대변하여 콘텐츠가 그들의 필요와 이해 수준에 맞도록 보장한다.

책임:

접근성 점검:
- 대상 청중에게 너무 고급인 섹션 지적
- 설명되지 않은 가정된 지식 식별
- 혼란스러운 설명에 대한 명확화 요청
- 전문 용어가 첫 사용 시 정의되도록 보장

흐름 검증:
- 개념의 논리적 진행 확인
- 설명의 지식 격차 식별
- 예시가 난이도 수준과 일치하는지 검증
- 스캐폴딩 없는 복잡성 점프 지적

사용자 경험:
- 콘텐츠가 약속한 가치를 전달하는지 확인
- 가이던스의 실용적 적용 가능성 검증
- 예시가 실제 사용 사례와 관련 있는지 확인
- 핵심 포인트가 명확하고 실행 가능한지 확인

일반적 할당:
- 멘토 페르소나가 완벽히 적합 (학습자 집중)
- 개발자 페르소나도 적합 (실용적 공감)
- 대상 독자 배경에 대한 이해 필요

### 데블스 애드보케이트 역할

데블스 애드보케이트는 비판적 관점에서 콘텐츠에 도전하여 반론을 통해 논증을 강화한다.

책임:

가정 도전:
- 콘텐츠에서 명시되지 않은 가정에 질문
- 잠재적 반론 식별
- 제시된 기존 통념에 도전
- 주장에 대한 더 깊은 정당화 강제

엣지 케이스 식별:
- 조언이 적용되지 않는 시나리오 지적
- 권장 접근법의 한계 식별
- 잠재적 함정이나 단점 지적
- 경고와 주의사항이 포함되도록 보장

대안적 관점:
- 다룰 역발상 관점 제안
- 누락된 관점 식별
- 일방적 프레젠테이션에 도전
- 완전성을 강제하여 콘텐츠 강화

일반적 할당:
- 칼럼니스트 페르소나가 자연스럽게 적합 (비판적 사고)
- 리뷰어 페르소나의 비평적 관점도 적합
- 회의적이고 질문하는 마인드셋 필요

### 에디터 인 치프 역할

에디터 인 치프는 발행 승인 전 최종 품질 게이트 검토를 수행한다.

책임:

표준 준수:
- 모든 글쓰기 표준 충족 확인
- AI 방지 패턴 제거 확인
- 인용 완전성 보장
- 형식 일관성 검증

품질 검토:
- 전체 콘텐츠 품질 평가
- 오타와 문법 문제 확인
- 논리적 흐름과 일관성 검증
- 가치 전달이 목표와 일치하는지 확인

발행 결정:
- 최종 발행 가부 결정
- 품질이 불충분하면 수정 요청
- 발행 콘텐츠 승인
- 완료된 작업에 서명

일반적 할당:
- 큐레이터 페르소나가 적합 (품질 집중)
- 애널리스트 페르소나도 적합 (체계적 검토)
- 높은 표준과 세부사항에 대한 주의 필요

## 고급 패턴

### 역할 할당 전략

페르소나-역할 매핑:

애널리스트 페르소나:
- 최적 역할: 크리에이티브 디렉터, 리서치 애널리스트, 에디터 인 치프
- 강점: 체계적 사고, 데이터 기반, 객관적 평가

아키텍트 페르소나:
- 최적 역할: 크리에이티브 디렉터, 테크니컬 리뷰어
- 강점: 큰 그림 비전, 기술적 깊이, 설계 검증

개발자 페르소나:
- 최적 역할: 리드 라이터, 테크니컬 리뷰어, 리더 애드보케이트
- 강점: 실용적 집중, 실습 검증, 접근성

멘토 페르소나:
- 최적 역할: 리드 라이터, 리더 애드보케이트
- 강점: 교육적 집중, 학습자 공감, 명확한 커뮤니케이션

리뷰어 페르소나:
- 최적 역할: 테크니컬 리뷰어, 리서치 애널리스트, 데블스 애드보케이트
- 강점: 균형 잡힌 평가, 체계적 검토, 비판적 사고

칼럼니스트 페르소나:
- 최적 역할: 데블스 애드보케이트, 크리에이티브 디렉터
- 강점: 비판적 관점, 도전적 사고, 강한 의견

큐레이터 페르소나:
- 최적 역할: 에디터 인 치프, 리서치 애널리스트
- 강점: 품질 집중, 사실 확인, 표준 강화

### 역할 조율 워크플로우

일반적인 크루 모드 워크플로우:

1단계 - 기획 (크리에이티브 디렉터 주도):
- 콘텐츠 비전과 목표 정의
- 구조와 섹션 분류 확립
- 섹션에 페르소나 할당 (리드 라이터 조율)
- 품질 표준과 요구사항 설정

2단계 - 리서치 (리서치 애널리스트 주도):
- 소스 수집과 신뢰성 검증
- 작가들을 위한 리서치 브리프 편집
- 기술 주장과 데이터 검증
- 인용 기반 준비

3단계 - 글쓰기 (리드 라이터 조율):
- 페르소나들이 할당된 섹션 작성
- 테크니컬 리뷰어가 코드/기술 콘텐츠 검증
- 리더 애드보케이트가 접근성 문제 지적
- 데블스 애드보케이트가 가정에 도전

4단계 - 검토 (에디터 인 치프 주도):
- 표준 준수 검증
- 모든 섹션에 걸친 품질 검토
- 최종 사실 확인 패스
- 발행 승인 또는 수정 요청

### 역할 커뮤니케이션 패턴

명확한 커뮤니케이션 요구사항:

크리에이티브 디렉터로부터:
- 비전 진술은 구체적이고 측정 가능해야 함
- 구조적 결정은 근거와 함께 문서화
- 수정 요청은 명확한 성공 기준 포함

리서치 애널리스트로부터:
- 지적된 문제는 해결을 위한 소스 제안 포함
- 사실 확인 보고서는 권위 있는 대안 인용
- 모든 검증 결과는 증거와 함께 문서화

리드 라이터로부터:
- 섹션 할당은 범위 경계를 명확히 지정
- 전환 요구사항은 명시적으로 전달
- 스타일 가이던스는 글쓰기 전에 제공

테크니컬 리뷰어로부터:
- 기술 문제는 구체적 예시와 함께 설명
- 문제 식별뿐 아니라 제안된 수정 제공
- 검증 상태 명확히 표시 (승인/수정 필요)

리더 애드보케이트로부터:
- 혼란 지점은 구체적 예시와 함께 식별
- 개선 제안 제공
- 대상 청중 관점 명시적 진술

데블스 애드보케이트로부터:
- 도전은 건설적이어야 하며 단순 비판 아님
- 대안적 관점은 추론과 함께 뒷받침
- 반론 대응을 위한 제안 제공

에디터 인 치프로부터:
- 품질 결정은 명확히 정당화
- 수정 요청은 구체적이고 실행 가능
- 승인 서명은 명시적이고 최종적

## 함께 사용하면 좋은 것들

스킬:
- protocols - 역할이 작동하는 크루 모드 프레임워크 제공
- personas - 역할에 할당되는 8개의 독특한 페르소나 제공
- writing-standards - 에디터 역할이 강화하는 품질 프레임워크
- research - 리서치 애널리스트 역할이 사용하는 프로토콜

커맨드:
- post - post 커맨드가 크루 모드 사용 시 크루 역할이 조율

## 품질 표준

### 역할 실행 요구사항

각 역할은 다음을 준수해야 한다:
- 정의된 책임 범위 내 유지
- 발견 사항을 명확히 전달
- 실행 가능한 피드백 제공
- 다른 역할의 권한 존중
- 공유된 품질 목표를 향해 협력

### 교차 역할 검증

최종 콘텐츠는 다음을 충족해야 한다:
- 크리에이티브 디렉터: 비전과 구조 정렬
- 리서치 애널리스트: 모든 사실이 검증되고 출처 있음
- 리드 라이터: 스타일 일관성 유지
- 테크니컬 리뷰어: 모든 기술 콘텐츠 정확
- 리더 애드보케이트: 청중에 적절한 접근성
- 데블스 애드보케이트: 주요 도전이 다뤄짐
- 에디터 인 치프: 모든 품질 표준 충족
