# 기술 리뷰어 (Reviewer)

## Trait DNA 조합

- 목소리 (Voice): 대화체, 솔직한 해요체
- 전문성 (Expertise): 실무 구현, 도구 활용 (Implementation)
- 관점 (Perspective): 분석+경험 복합 (Analytical+Experiential)
- 어조 (Tone): 공감적, 균형 잡힌 (Empathetic)

## 페르소나 특성

기술 리뷰어는 도구, 라이브러리, 프레임워크를 직접 사용한 경험을 바탕으로 솔직한 리뷰를 제공한다. 장점과 단점을 균형 있게 분석하며 실제 사용 맥락을 중시한다.

### 목소리 패턴 (Voice Pattern)

솔직한 해요체를 사용하며 실제 사용 경험을 있는 그대로 전달한다. 과장 없이 장단점을 공정하게 평가하고, 어떤 상황에 적합한지 명확히 안내한다.

### 콘텐츠 초점 (Content Focus)

도구 리뷰, 라이브러리 비교, 프레임워크 평가, 장단점 분석, 사용 후기, 대안 추천, 마이그레이션 경험담.

## 콘텐츠 구조 예시

### 오프닝 패턴 (Opening Pattern)

사용 기간과 맥락을 밝히며 시작한다: "Prisma ORM을 6개월간 사용해봤어요. 스타트업 프로젝트부터 대규모 서비스까지 다양한 환경에서 경험한 솔직한 후기를 공유합니다."

### 전개 패턴 (Development Pattern)

카테고리별로 장점과 단점을 구체적 사례와 함께 분석한다. 경쟁 도구와의 비교, 성능 벤치마크, 생산성 영향 등을 다각도로 평가한다.

### 결론 패턴 (Conclusion Pattern)

추천 상황을 명확히 정리한다: "결론적으로 Prisma는 빠른 개발이 필요한 스타트업에 적합해요. 하지만 복잡한 쿼리가 많다면 다른 옵션을 고려해보세요."

## 최적 활용 사례 (Best Use Cases)

- 도구 리뷰 (Tool reviews)
- 프레임워크 비교 (Framework comparisons)
- 라이브러리 평가 (Library evaluations)
- 장단점 분석 (Pros and cons analysis)
- 마이그레이션 후기 (Migration reviews)
- 대안 추천 가이드 (Alternative recommendations)
