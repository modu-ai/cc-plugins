# 리서치 프로토콜 (Research Protocol)

## 빠른 참조

### 목적

리서치 프로토콜은 소스 검증, 미디어 수집, 인용 관리를 포함하여 aStory 블로그 콘텐츠 생성을 위한 포괄적인 웹 리서치 기능을 제공한다.

### 핵심 기능

다단계 웹 검색 (Multi-Stage Web Search):
- 핵심 주제를 직접 타겟팅하는 1차 검색
- 관련 개념으로 확장하는 2차 검색
- 특정 기술 세부사항을 타겟팅하는 3차 검색
- 주요 주장을 확인하는 검증 검색

미디어 수집 (Media Collection):
- 라이선스 추적이 있는 검증된 소스에서 이미지 추출
- 타임스탬프 분석이 있는 YouTube 동영상 수집
- 콘텐츠 섹션에 리소스를 매핑하는 사용 계획

소스 검증 (Source Verification):
- WebFetch를 통한 URL 접근성 확인
- 콘텐츠 최신성 검증 (6개월 이상된 소스에 대한 업데이트)
- 여러 소스에서 사실 정확성 교차 검증
- 저자 신뢰도 평가

### 리서치 브리프 출력

리서치 프로세스는 다음을 포함하는 구조화된 브리프를 생성한다:
- 주요 발견 사항이 포함된 주제 요약
- URL, 유형, 관련성 등급이 있는 번호 매겨진 소스 목록
- URL, 설명, 라이선스 정보가 있는 이미지 컬렉션
- YouTube 링크, 타임스탬프, 사용 계획이 있는 동영상 컬렉션
- 각 콘텐츠 섹션을 지원하는 리소스를 보여주는 섹션 매핑
- 참조 섹션을 위한 형식화된 인용

## 구현 가이드

### 리서치 워크플로우

리서치는 광범위한 검색에서 구체적인 리소스 할당으로 구축되는 5단계 프로세스를 따른다.

1단계 - 주제 분석:
- 핵심 개념을 식별하기 위해 주제 검토
- 관련 기술과 권위 있는 소스 식별
- 콘텐츠 유형에 기반한 검색 전략 결정

2단계 - 웹 검색 실행:
- 다양한 쿼리 구성으로 다중 검색 수행
- 주제의 다른 측면 타겟팅
- 결과 수집 및 중복 제거

3단계 - 소스 검증:
- 각 잠재적 소스를 신뢰성, 최신성, 관련성에 대해 평가
- 검증된 소스만 수집 단계로 진행
- 소스 유형 추적 및 공식 문서 우선

4단계 - 미디어 수집:
- 검증된 소스에서 이미지와 YouTube 동영상 추출
- 모든 미디어의 라이선스 정보 추적
- 맥락과 계획된 사용 문서화

5단계 - 브리프 편집:
- 모든 리소스를 구조화된 브리프로 정리
- 리소스 할당이 있는 섹션 매핑 생성
- 참조 섹션을 위한 인용 형식화

### 소스 우선순위 계층

공식 문서 (최우선):
- Anthropic 문서, 공식 GitHub 저장소, 회사 블로그 포스트
- 가장 권위 있고 정확한 소스

확립된 기술 출판물 (2순위):
- 인정받는 기술 블로그, 컨퍼런스 논문, 동료 검토 콘텐츠
- 편집 감독이 있는 강한 신뢰성

커뮤니티 리소스 (3순위):
- 높은 점수의 Stack Overflow 답변
- 공식 응답이 있는 GitHub 이슈
- 정확성이 검증된 튜토리얼 사이트

개인 블로그 포스트 (4순위):
- 포함 전 추가 검증 필요
- 가치 있는 통찰이 있을 수 있지만 교차 검증 필요

뉴스 기사 (5순위):
- 맥락을 제공하지만 기술적 깊이는 드뭄
- 타임라인과 산업 맥락에 사용

### 검증 요구사항

모든 소스는 포함 전 검증 기준을 충족해야 한다:

URL 접근성:
- WebFetch 도구가 각 URL이 유효한 콘텐츠를 반환하는지 확인
- 깨진 링크는 폐기하거나 대체

콘텐츠 최신성:
- 6개월 이상된 소스는 업데이트된 버전 확인
- 기술 버전과 릴리스 날짜 검증

사실 정확성:
- 주요 주장은 최소 하나의 추가 소스에 나타나야 함
- 통계와 벤치마크 교차 참조

저자 신뢰도:
- 알려진 전문가와 공식 대표자에 더 높은 가중치
- 익명 또는 미지의 저자는 더 강한 교차 검증 필요

상세한 검색 프로토콜은 [web-search-protocol.md](modules/web-search-protocol.md) 참조.

## 고급 패턴

### 이미지 수집 프로토콜

이미지 수집은 적절한 미디어 사용을 보장하기 위한 특정 가이드라인을 따른다:

우선 이미지:
- 공식 문서의 아키텍처 다이어그램
- 기능을 보여주는 주석이 달린 스크린샷
- 개념을 설명하는 인포그래픽
- 비교 시각화

라이선스 추적:
- 크리에이티브 커먼즈 라이선스 선호
- 미지의 라이선스는 직접 이미지 파일이 아닌 소스 페이지를 가리키는 이미지 URL 필요
- 각 이미지의 라이선스 조건 문서화

맥락 문서화:
- 이미지가 어디서 발견되었는지 설명
- 무엇을 묘사하는지 설명
- 콘텐츠를 어떻게 지원하는지 명확화
- 적절한 alt 텍스트와 캡션 생성 가능하게 함

사용 계획:
- 각 이미지를 특정 콘텐츠 섹션에 할당
- 동일 위치에 여러 이미지가 경쟁하지 않도록 방지
- 콘텐츠 전반에 걸쳐 이미지 분포 균형

상세한 미디어 수집 절차는 [media-collection.md](modules/media-collection.md) 참조.

### YouTube 수집 프로토콜

YouTube 동영상은 적절히 큐레이션될 때 가치 있는 보충 콘텐츠를 제공한다:

우선 동영상:
- 공식 제품 데모
- 인정받는 전문가의 컨퍼런스 발표
- 관련 기능의 튜토리얼 워크스루
- 확립된 채널의 기술 딥다이브

타임스탬프 분석:
- 주제와 관련된 특정 세그먼트 식별
- 전체 동영상보다 타임스탬프 범위 기록
- 독자가 관련 섹션으로 점프할 수 있게 함

임베드 계획:
- 동영상 임베드에 YouTubeCard 컴포넌트 사용
- 각 동영상을 특정 콘텐츠 위치에 할당
- 동영상이 섹션을 지원하는 이유를 설명하는 맥락 제공

### 리서치 브리프 구조

리서치 브리프는 다운스트림 에이전트가 파싱 가능한 표준화된 형식을 따른다:

수집 요약:
- 주제 식별자
- 수집된 소스, 이미지, 동영상 수
- 검색 전략 개요

소스 섹션:
- ID, URL, 제목이 있는 각 검증된 소스
- 유형 분류 (공식, 출판물, 커뮤니티 등)
- 관련성 등급 (높음, 중간, 낮음)
- 콘텐츠에서 추출된 핵심 포인트

이미지 섹션:
- URL, alt 텍스트가 있는 각 수집된 이미지
- 맥락 설명
- 계획된 사용 위치
- 라이선스 정보

동영상 섹션:
- URL, 제목, 길이가 있는 각 YouTube 동영상
- 관련 세그먼트를 위한 타임스탬프 노트
- 계획된 사용과 근거

섹션 매핑:
- 콘텐츠 구조에 리소스 할당
- 어떤 소스가 도입부를 지원하는지 보여줌
- 미디어 임베드가 어디에 나타나야 하는지 표시

인용 섹션:
- 참조 섹션을 위한 사전 형식화된 항목
- 일관된 인용 표준 따름

## 품질 표준

### 소스 검증 요구사항

모든 소스는 다음 기준을 충족해야 한다:
- URL이 WebFetch를 통해 유효하고 접근 가능한 콘텐츠 반환
- 콘텐츠 최신성이 논의되는 기술 버전에 적절
- 사실이 최소 하나의 추가 독립 소스에서 교차 검증됨
- 저자 신뢰도가 평가되고 문서화됨

### 미디어 검증 요구사항

이미지:
- 모든 이미지 URL이 유효하고 접근 가능
- 라이선스 정보가 완전히 문서화됨
- 맥락 설명이 전체 세부사항 제공
- 사용 계획이 특정 섹션에 할당됨

동영상:
- 모든 YouTube URL이 유효하고 접근 가능
- 타임스탬프가 정확하고 관련됨
- 임베드 위치가 적절히 계획됨

### 브리프 완전성 요구사항

섹션 매핑:
- 모든 주요 콘텐츠 섹션에 할당된 리소스 있음
- 섹션 할당 없는 고아 리소스 없음
- 인용이 적절히 형식화됨

브리프 구조:
- 구조화된 형식 사용 시 YAML 구문 유효
- 모든 필수 섹션 존재
- 카운트가 실제 수집된 항목과 일치
