# 글쓰기 표준 (Writing Standards)

## 빠른 참조

### 목적

글쓰기 표준 스킬은 AI 패턴 제거, 전지적 저자 관점, 체계적인 인용 관행을 통해 고품질 한국어 블로그 글쓰기를 강화한다.

### 핵심 원칙

전지적 저자 시스템 (Omniscient Author System):
- 모든 콘텐츠는 격식체 선언형 스타일로 3인칭 관점 사용
- 작가는 전지적 서술자로서 관찰하고 설명
- 1인칭이나 2인칭 호칭 없음
- 전체적으로 객관적 분석적 입장

AI 패턴 제거 (AI Pattern Elimination):
- 특징적인 AI 생성 표현 식별 및 대체
- 가장 중요: "~를 넘어 ~하다" 구문 완전 제거
- 금지 어휘와 공허한 강조어 제거
- 반복을 피하기 위한 어휘 다양화 강화

인용 시스템 (Citation System):
- 모든 사실적 주장은 본문에 번호 인용 필요
- 각 장 끝에 완전한 참조 섹션
- 신뢰성과 최신성을 위한 소스 검증

### 절대 금지 사항

금지 어휘:
- 추상적 과용 용어: "패러다임", "혁신", "획기적", "놀라운"
- 공허한 강조어: "매우", "아주", "정말", "물론", "당연히"
- 채움 표현: "현저하게", "상당히", "분명히", "확실히", "명백히"

금지 패턴:
- 어떤 형태로든 "~를 넘어 ~하다" 구문
- 추측형 어미: "~일 것입니다", "~로 예상됩니다", "~로 보입니다"
- 2인칭 호칭: "사용자", "여러분", "당신"

### 스타일 요구사항

문장 종결:
- 전체적으로 격식체 선언형 "한다체" 스타일
- 서술형: "~다", "~한다", "~된다"
- 의문형: "~하는가", "~인가"
- 명령형: "~하자", "~해보자"

단락 구조:
- 단락당 최소 3문장
- 본문에 글머리 기호나 번호 목록 없음 (흐르는 서술 사용)
- 단락 간 논리적 연결어

대상 청중:
- 1-3년 경력의 주니어 개발자
- 기술 용어는 첫 사용 시 한국어 번역 + 영어 괄호 표기 필요

완전한 표준은 아래 모듈 파일 참조.

## 구현 가이드

### 전지적 저자 시스템

전지적 저자 시스템은 엄격한 3인칭 준수로 모든 aStory 콘텐츠의 서술 관점을 확립한다.

1인칭 금지:
- "저", "나" 또는 어떤 1인칭 구문도 절대 사용 금지
- 잘못된 예: "저는 Claude Code를 설명하겠습니다"
- 올바른 예: "Claude Code는 다음과 같은 특징을 가진다"

2인칭 금지:
- "사용자", "여러분", "당신"으로 독자를 직접 호칭 금지
- 잘못된 예: "사용자는 이 기능을 사용해야 합니다"
- 올바른 예: "개발자는 이 기능을 사용한다"

객관적 입장:
- 작가는 관찰적, 분석적 자세 유지
- 주관적 의견과 개인적 반응 배제
- 기술 콘텐츠를 개인적 해석이 아닌 확립된 사실로 제시

### 격식체 선언형 스타일

모든 문장은 "한다체" 격식체 선언형 어조를 따른다:

서술형 어미:
- "~다", "~한다", "~된다" 사용
- 예시: "Claude Code는 터미널 기반의 도구이다"

의문형 어미:
- "~하는가", "~인가" 사용
- 예시: "어떤 점에서 다른가"

명령형 어미:
- "~하자", "~해보자" 사용
- 예시: "다음에서 자세히 살펴보자"

### 다중 단락 서술 형식

콘텐츠는 목록이나 글머리 기호가 아닌 흐르는 단락으로 구성된다:

단락 요구사항:
- 각 단락은 최소 3문장 포함
- 첫 문장은 주제 도입
- 중간 문장은 설명과 증거 제공
- 마지막 문장은 다음 포인트로 전환하거나 단락 요약

논리적 연결어:
- 단락을 전환으로 연결: "그러나", "반면에", "이에 따라", "결국"

개념 전개:
- 핵심 개념을 3-7개 단락에 걸쳐 설명
- 각 단락은 개념의 한 측면을 다룸
- 점진적으로 이해 구축

상세한 패턴과 예시는 지원 모듈 파일 참조.

## 고급 패턴

### 어휘 다양화

반복 감지:
- 단일 단락 내에서 동일 단어가 두 번 이상 나타나지 않음
- 작가는 정신적 어휘 추적기 유지
- 동의어를 체계적으로 순환

일반적 순환:
- "기능" 대안: "역할", "책임", "임무", "능력"
- "제공" 대안: "지원", "공급", "제시", "제안"
- "처리" 대안: "다루다", "관리", "실행", "수행"

### 문장 구조 변화

능동/수동 교체:
- 능동태와 수동태 사이 교체
- 능동: "시스템이 데이터를 처리한다"
- 수동: "데이터가 시스템에 의해 처리된다"

길이 변화:
- 짧은 문장 (15-20단어): 핵심 포인트 전달
- 중간 문장 (25-35단어): 상세한 설명 제공
- 긴 문장 (40-50단어): 복잡한 기술 관계 처리

### 연결어 다양성

순차적 연결어:
- "그리고", "또한", "그뿐만 아니라"

대조적 연결어:
- "그러나", "반면에", "그럼에도 불구하고"

인과적 연결어:
- "그래서", "따라서", "이에 따라", "결론적으로"

전환적 연결어:
- "한편", "다른 한편으로", "또한"

## 품질 표준

### 제출 전 검증

AI 패턴 점검:
- 어디에도 "~를 넘어" 패턴 없음
- 금지 어휘 나타나지 않음
- 추측 표현 없음
- 단일 단락 내에서 동일 단어 두 번 반복 없음

스타일 점검:
- 전체적으로 3인칭 관점 유지
- 격식체 선언형 "한다체" 스타일 일관되게 사용
- 단락당 최소 3문장
- 단락 간 논리적 연결어 존재

인용 점검:
- 모든 통계와 사실에 인용 번호 있음
- 장 끝에 참조 섹션 나타남
- 인용 번호가 참조 목록과 일치

### 검증 체크리스트

콘텐츠는 모든 요구사항을 충족해야 한다:
- AI 패턴 완전히 제거됨
- 스타일 일관성 유지됨
- 인용 완전성 검증됨
- 어휘 다양성 확인됨
- 문장 구조 적절히 변화됨
- 연결어 사용이 적절히 분포됨
